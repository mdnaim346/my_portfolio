# Naim Reza Portfolio

Advanced personal portfolio for an Odoo Full Stack Developer. It is built with React, Framer Motion, CSS-driven smart project visuals, and a small Express server for project data and production preview.

## What This Portfolio Includes

- Responsive hero section with profile image from `public/profile.png`
- Dynamic animated page background
- Project showcase with category filters
- Smart project visuals generated from each project name/category
- Selected case-study panel with impact, stack, and role details
- Skills, process, and contact sections
- Express API for project data at `/api/projects`
- Production build served by `server.js`

## How It Works

The main UI lives in `src/App.js`.

Project cards are loaded from two places:

1. The app first uses built-in fallback project data inside `src/App.js`.
2. If the Express server is running, it fetches live project data from:

```text
http://localhost:5001/api/projects
```

That API reads from:

```text
projects.json
```

If the API is not running, the portfolio still works because the fallback project data is already inside the React app.

## Smart Project Images

The project images are not normal JPG thumbnails. They are generated by React and CSS from each project's:

- `title`
- `category`
- `stack`

The component is:

```text
SmartProjectImage
```

inside:

```text
src/App.js
```

The styling is in:

```text
src/App.css
```

This means if you add a new project name like `Hospital ERP System`, the card automatically creates a matching visual using the title initials and category theme.

## Update Your Profile

Profile content is in `src/App.js`:

```js
const profile = {
  name: 'Naim Reza',
  role: 'Odoo Full Stack Developer',
  headline: 'I design ERP systems that make complex operations feel clear, fast, and measurable.',
  location: 'Dhaka, Bangladesh',
  email: 'naim.reza@example.com',
  github: 'https://github.com/naimreza',
  linkedin: 'https://linkedin.com/in/naimreza',
};
```

Replace the email, GitHub, and LinkedIn values with your real links.

Your profile image is loaded from:

```text
public/profile.png
```

To change it, replace that file with another image using the same name.

## Add Or Edit Projects

Edit:

```text
projects.json
```

Each project should look like this:

```json
{
  "id": "sale-approval",
  "title": "Sale Order Approval Workflow",
  "category": "ERP",
  "desc": "Multi-level approval flows for Odoo sales teams with automated notifications and audit-ready status tracking.",
  "image": "/project1.jpg",
  "impact": "Reduced approval delays by 42%",
  "stack": ["Odoo", "Python", "PostgreSQL", "XML", "JavaScript"],
  "role": "Architecture, module development, testing, deployment"
}
```

Important fields:

- `id`: unique project id, use lowercase words with hyphens
- `title`: shown on the card and used for the smart visual
- `category`: used for filtering and visual theme
- `desc`: short project description
- `impact`: business result or project value
- `stack`: technologies used
- `role`: what you did in the project

## Run In Development

Install dependencies first:

```bash
npm install
```

Start the React development server:

```bash
npm start
```

Open:

```text
http://localhost:3000
```

If PowerShell blocks `npm`, use:

```bash
npm.cmd start
```

## Run The API Server

In another terminal:

```bash
npm run server
```

Or on PowerShell:

```bash
npm.cmd run server
```

The API runs at:

```text
http://localhost:5001/api/projects
```

## Production Preview

Build the React app:

```bash
npm run build
```

Then serve the production build and API together:

```bash
npm run server
```

Open:

```text
http://localhost:5001
```

This works because `server.js` serves both:

- `/api/projects`
- the optimized React files from `build/`

## Main Files

- `src/App.js`: main portfolio UI and content logic
- `src/App.css`: all layout, smart visuals, responsive design, and dynamic background
- `projects.json`: live project data for the Express API
- `server.js`: Express server for API and production build
- `public/profile.png`: profile image
- `public/index.html`: SEO title and meta description

## Deployment Notes

For a simple deployment, run:

```bash
npm run build
```

Then deploy the `build/` folder to a static host.

If you want the live `/api/projects` endpoint, deploy the full Node/Express app instead and run:

```bash
npm run server
```
